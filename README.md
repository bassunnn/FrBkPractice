# Контрольная работа №3
Практики 13-17

## Практическая работа №13

**Что реализовано:**

- Разработано веб-приложение для управления списком заметок (менеджер задач).
- Реализованы базовые CRUD-операции с заметками (просмотр, добавление) с сохранением данных в `localStorage`.
- Создан и зарегистрирован Service Worker (`sw.js`), который перехватывает сетевые запросы (`fetch`).
- Реализовано кэширование статических ресурсов (HTML, CSS, JS) в событии `install` Service Worker.
- Настроена стратегия Cache First: при отсутствии сети страница и скрипты загружаются из кэша, интерфейс остается полностью работоспособным офлайн.

**Технологии:** JavaScript (Vanilla), Service Worker API, Cache API, localStorage.

## Практическая работа №15

**Что реализовано:**

- Обеспечена безопасность приложения с помощью локального HTTPS: установлена утилита `mkcert`, сгенерированы самоподписанные сертификаты (`localhost.pem`, `localhost-key.pem`).
- Сервер запущен по протоколу HTTPS (через `http-server` с ключами SSL) на порту 3000.
- Реализована архитектура App Shell (каркас приложения): базовая структура (`index.html`) содержит только шапку, навигацию и основной контейнер.
- Динамический контент (страницы "Главная" и "О приложении") вынесен в отдельные HTML-файлы в папку `content/` и загружается через `fetch()` при навигации.
- Модифицирован Service Worker: реализована стратегия Network First для динамических страниц (`/content/*`) и Cache First для статических ресурсов (App Shell).
- Добавлена страница "О приложении" с информацией, которая корректно кэшируется и доступна офлайн.
- Проверена работа: при отключении сети каркас отображается мгновенно, контент подгружается из кэша, добавление заметок работает офлайн.

**Технологии:** HTTPS (mkcert), App Shell Architecture, Service Worker (Cache First, Network First), Fetch API.

## Практическая работа №16

**Что реализовано:**

- Реализован полноценный WebSocket-сервер на Node.js с использованием библиотеки `socket.io` и `express`.
- Сгенерированы VAPID-ключи для отправки push-уведомлений (`web-push generate-vapid-keys`).
- Создан сервер (`server.js`), который раздает статику, обрабатывает WebSocket-соединения, хранит push-подписки в массиве и предоставляет эндпоинты `/subscribe` и `/unsubscribe`.
- В клиентской части (файлы `index.html`, `app.js`, `sw.js`) интегрирована библиотека `socket.io/client`.
- Реализована отправка события `newTask` на сервер при добавлении новой задачи.
- Реализован прием события `taskAdded` от сервера и отображение всплывающего уведомления в интерфейсе.
- Добавлена возможность подписки на push-уведомления через `PushManager` (функции `subscribeToPush`, `unsubscribeFromPush`), с отправкой подписки на сервер.
- При добавлении задачи сервер через `io.emit()` рассылает событие всем клиентам WebSocket и инициирует отправку push-уведомлений всем подписанным клиентам.
- Добавлены кнопки интерфейса "Включить уведомления" / "Отключить уведомления".
- Модифицирован Service Worker: добавлен обработчик события `push` для отображения системных уведомлений.
- Проверена работа: при добавлении задачи во второй вкладке сразу появляется WebSocket-уведомление; push-уведомление приходит даже при закрытом/неактивном приложении.

**Технологии:** Node.js, Express, Socket.IO, Web-Push, VAPID, Service Worker (Push API), localStorage.

## Практическая работа №17

**Что реализовано:**

- Доработано приложение из практики №16: добавлен функционал отложенных напоминаний (reminders).
- В клиентской части (`content/home.html`) добавлена форма для создания заметки с напоминанием: текстовое поле + `datetime-local` (выбор даты и времени).
- Изменена структура хранения заметок в `localStorage`: каждая заметка содержит уникальный `id` (Date.now()), текст `text` и `reminder` (timestamp времени напоминания).
- В `app.js` добавлена обработка формы напоминаний: при отправке создается объект заметки и отправляется событие `newReminder` на сервер через Socket.IO.
- На сервере (`server.js`) создано хранилище активных напоминаний (`Map`), где ключ — `id` заметки, значение — таймер (`setTimeout`) и данные.
- При получении события `newReminder` сервер вычисляет задержку (`delay = reminderTime - Date.now()`) и устанавливает таймер. По истечении таймера отправляется push-уведомление всем подписанным клиентам.
- Реализован эндпоинт `/snooze` (POST), который принимает `reminderId`, отменяет предыдущий таймер и устанавливает новый на 5 минут (откладывание напоминания).
- В Service Worker добавлен обработчик `notificationclick`: при нажатии на кнопку "Отложить на 5 минут" отправляется запрос на сервер (`/snooze`). При нажатии на само уведомление — оно закрывается.
- В обработчик `push` добавлена передача `reminderId` в `event.data` и опция `actions` с кнопкой "Отложить на 5 минут" для уведомлений-напоминаний.
- Проверена работа: напоминания приходят в указанное время, есть возможность отложить их на 5 минут через кнопку в уведомлении, напоминания работают даже при закрытом приложении.

**Технологии:** Node.js, Express, Socket.IO, Web-Push, Service Worker (Push API, Notification API), `setTimeout`, `localStorage`.
